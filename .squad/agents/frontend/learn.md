# Frontend Agent — Codebase Learning Protocol

When instructed to learn the codebase, execute these steps IN ORDER.
Save your findings to `.copilot/skills/role-frontend-core.md` using the skill bundle format.

---

## Step 1: Identify the Stack

Check for frontend framework indicators:
```
# Framework detection
ls package.json 2>/dev/null && cat package.json | head -100
ls angular.json next.config.* nuxt.config.* vite.config.* webpack.config.* svelte.config.* 2>/dev/null
ls tsconfig.json .babelrc tailwind.config.* postcss.config.* 2>/dev/null
```

Identify:
- **Framework:** (React, Angular, Vue, Svelte, Next.js, Nuxt, etc.)
- **Language:** (TypeScript, JavaScript)
- **Build tool:** (Vite, Webpack, Turbopack, esbuild, Nx, etc.)
- **Styling:** (Tailwind, CSS Modules, Styled Components, SCSS, Material, etc.)
- **State management:** (Redux, Zustand, Pinia, signals, Context, etc.)
- **UI library:** (Material UI, shadcn, Radix, PrimeNG, etc.)
- **Monorepo?** (Nx, Turborepo, pnpm workspaces?)

## Step 2: Map the Frontend Directory Structure

```
# Find frontend source files
find . -type f \( -name "*.tsx" -o -name "*.jsx" -o -name "*.vue" -o -name "*.svelte" -o -name "*.component.ts" \) -not -path '*/node_modules/*' -not -path '*/dist/*' | head -60
```

Document:
- Where components live (flat? feature folders? atomic design?)
- Where pages/routes live
- Where services/API calls live
- Where types/interfaces live
- Where hooks/composables/utilities live
- Where styles live
- Where tests live

## Step 3: Find the Reference Implementation

Look for the most complete feature — one that has:
- A page/route
- Components
- API integration (service/hook/composable)
- Types/interfaces
- Tests (if any)

Read the most complete feature thoroughly.

## Step 4: Extract Component Patterns

Read 3-5 components and document:

### Component Structure
- Functional or class components?
- Props definition style (interface? type? inline?)
- State management inside components (useState? signals? reactive?)
- Event handling patterns
- How is DI/injection done? (hooks? inject()? props?)

### Routing
- How are routes defined? (file-based? config-based?)
- Lazy loading? Code splitting?
- Route guards/middleware?
- Nested routes pattern?

### API Integration
- How are API calls made? (fetch? axios? HttpClient? custom hook?)
- Base URL configuration?
- Error handling in API calls?
- Loading/error state management?
- Request/response types — how do they match backend DTOs?

### Styling
- Approach (CSS modules? Tailwind? styled-components? SCSS?)
- Naming conventions for classes
- Theme/design system usage
- Responsive patterns

### Form Handling
- Form library? (React Hook Form, Formik, Angular Reactive Forms?)
- Validation approach?
- Error display patterns?

## Step 5: Extract Type/Interface Conventions

- How are shared types organized?
- DTO interfaces — do they match backend exactly?
- Enum vs union type usage?
- Generic patterns?
