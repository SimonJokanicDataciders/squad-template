# Lead Agent — Codebase Learning Protocol

When instructed to learn the codebase, execute these steps IN ORDER.
Save your findings to `.copilot/skills/role-architect.md` AND `.copilot/skills/sdlc-context-core.md`.

---

## Step 1: High-Level Architecture Scan

```
# Get the full directory tree (top 3 levels)
find . -maxdepth 3 -type d -not -path '*/node_modules/*' -not -path '*/.git/*' -not -path '*/dist/*' -not -path '*/__pycache__/*' -not -path '*/build/*' -not -path '*/.next/*' | sort

# Check for monorepo indicators
ls lerna.json nx.json turbo.json pnpm-workspace.yaml 2>/dev/null

# Check for infrastructure/deploy
ls docker-compose* Dockerfile* .github/workflows/* Procfile serverless.* pulumi* terraform* k8s/ 2>/dev/null
```

Identify:
- **Architecture:** (monolith? microservices? monorepo? serverless?)
- **Layers:** (frontend + backend? API only? full-stack?)
- **Infrastructure:** (Docker? Kubernetes? serverless? PaaS?)
- **CI/CD:** (GitHub Actions? GitLab CI? Jenkins? CircleCI?)

## Step 2: Dependency & Stack Analysis

Read ALL package/project files:
```
# Find all dependency manifests
find . -maxdepth 3 \( -name "package.json" -o -name "requirements*.txt" -o -name "pyproject.toml" -o -name "*.csproj" -o -name "go.mod" -o -name "Cargo.toml" -o -name "Gemfile" -o -name "pom.xml" -o -name "build.gradle" \) -not -path '*/node_modules/*' | head -10
```

For each, document:
- Key runtime dependencies
- Key dev dependencies
- Build scripts / commands
- Version constraints

## Step 4: Identify the Reference Implementation

Find the most complete feature that spans all layers:
- Has an entity/model
- Has a service/business logic
- Has API endpoints
- Has frontend UI
- Has tests

This is the feature ALL agents should follow.

## Step 5: Analyze Development Workflow

```
# Check git history for conventions
git log --oneline -20 2>/dev/null

# Check for contribution guidelines
cat CONTRIBUTING.md .github/CONTRIBUTING.md 2>/dev/null | head -50

# Check for linting / formatting
ls .eslintrc* .prettierrc* .editorconfig .rubocop.yml .pylintrc ruff.toml 2>/dev/null
```

Document:
- Commit message convention (conventional commits? free-form?)
- Branch strategy (git flow? trunk-based? feature branches?)
- PR process
- Linting / formatting tools

## Step 6: Analyze Build & Run Commands

Document the exact commands for:
- Install dependencies
- Start development server
- Build for production
- Run tests
- Run linting
- Database migrations
